Self-host on a Mac (native)

The setup the author runs at home: a Mac mini in a cupboard, the server as a background service, local AI on Apple silicon. About fifteen minutes.

last updated 14 Jun 2026 · applies to server 1.x
note

Prefer containers? The Linux guide uses Docker Compose and works on a Mac too. This page is for the native install, which is faster on Apple silicon.

What you need

Any Mac with Apple silicon works. The archive itself is small — the letters of a household for a decade fit in a few gigabytes; the AI models take the most room.

Install

One command fetches the server, the models, and registers it as a service that starts with the Mac:

curl -fsSL https://get.briefkist.eu/mac | sh

When it finishes, the server prints its address on your network and an admin URL:

briefkist server 1.4.2 — ready
archive:  /Users/you/Briefkist
address:  http://brievenbus.local:8420
admin:    http://brievenbus.local:8420/setup
caution

During setup you choose the passphrase that encrypts the archive. There is no reset. Write it down before you continue — the setup screen won't let you past without confirming you have.

Pair your phone

Open the admin URL on the Mac, name the device ("Anna's phone"), and press "Create pairing code". Scan the code with the app. This phone will talk only to your own server — the token it receives works nowhere else, and you can revoke it from the settings screen at any time.

File your first letter

Photograph any letter from today's pile. Within seconds the archive shows the sender, the date, a summary and keywords — all extracted on the Mac in the cupboard. Nothing left the house.
